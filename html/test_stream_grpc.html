🎬 gRPC Video Streaming Demo

Architecture microservices inspirée des plateformes modernes de streaming

⚙️ Comment fonctionne le streaming

Lorsque cette page charge la vidéo, le navigateur demande :

/video/goldorak/index.m3u8

Ce fichier est la playlist HLS utilisée pour le streaming adaptatif.

1️⃣ Vérification du cache

  • ✅ Si le HLS existe → lecture immédiate
  • ❌ Sinon → génération automatique

2️⃣ Téléchargement dynamique via gRPC

Si la vidéo n’existe pas encore, le Video Gateway télécharge la vidéo originale depuis un microservice gRPC.

La vidéo est reconstruite chunk par chunk grâce à un streaming binaire gRPC :

VideoService.StreamVideo()

👉 Le fichier vidéo exact est donc récupéré dynamiquement avant toute conversion.

3️⃣ Normalisation FFmpeg

La vidéo source (AVI, MPG, etc.) est automatiquement nettoyée :

  • correction timestamps
  • conversion codecs
  • compatibilité navigateur

4️⃣ Génération HLS

FFmpeg segmente ensuite la vidéo :

index.m3u8 + seg_000.ts → seg_XXX.ts

Chaque segment ≈ 4 secondes, permettant un démarrage instantané.

5️⃣ Lecture progressive

Le navigateur télécharge uniquement les segments nécessaires, exactement comme Netflix ou Disney+.

🧠 Schéma global de l’architecture

Pipeline complet :

  • Browser → demande HLS
  • Gateway → vérifie cache
  • Si absent → download vidéo via gRPC
  • FFmpeg → normalise + segmente
  • HLS → streaming progressif
